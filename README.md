# Система бронирования переговорных комнат

## Описание проекта

Проект посвящен описанию и проектированию системы бронирования переговорных комнат для сотрудников компании.

Система позволяет сотрудникам просматривать список переговорных комнат, проверять их доступность, создавать бронирования и отменять свои бронирования.

Администратор системы может управлять переговорными комнатами: добавлять новые комнаты, редактировать информацию о них и отключать комнаты, которые временно недоступны.

## Цель проекта

Цель проекта — изучить основные навыки системного аналитика на примере одной системы:

- описание требований;
- формулирование бизнес-правил;
- моделирование бизнес-процессов;
- создание UML-диаграмм;
- проектирование базы данных;
- описание REST API;
- создание простого технического прототипа.

## Цель системы

Цель системы — позволить сотрудникам компании удобно бронировать переговорные комнаты, проверять их доступность и снижать количество конфликтов при использовании помещений.

## Пользователи системы

В системе предполагаются следующие роли:

- сотрудник;
- администратор;
- система уведомлений.

## Основные возможности

В первой версии система должна позволять:

- просматривать список переговорных комнат;
- просматривать информацию о конкретной комнате;
- создавать бронирование комнаты;
- отменять бронирование;
- проверять доступность комнаты;
- управлять комнатами со стороны администратора.

## Роль системного аналитика в проекте

В рамках проекта системный аналитик отвечает за описание того, как должна работать система бронирования переговорных комнат.

Основные задачи системного аналитика в этом проекте:

- определить пользователей системы и их роли;
- описать проблему, которую должна решить система;
- зафиксировать границы первой версии;
- описать функциональные и нефункциональные требования;
- сформулировать бизнес-правила;
- описать основные пользовательские сценарии;
- подготовить диаграммы бизнес-процессов и взаимодействий;
- спроектировать модель данных;
- описать REST API;
- согласовать логику системы между бизнесом, разработкой и тестированием.

Системный аналитик в этом проекте связывает бизнес-идею с технической реализацией.

## Мое понимание роли системного аналитика

Системный аналитик — это специалист, который помогает превратить бизнес-запрос в понятное описание IT-системы. Его задача — разобраться в потребностях бизнеса, уточнить требования, описать логику работы системы и зафиксировать это так, чтобы разработчики, тестировщики и другие участники команды одинаково понимали, что нужно реализовать. В проекте бронирования переговорных комнат системный аналитик должен понять, как сотрудники будут бронировать комнаты, какие ограничения есть у системы и какие данные нужно хранить.
